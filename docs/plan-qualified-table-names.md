# Adopt avrotize --namespace / --qualified-table-names

## Why
`tools/pugetsound/fabric/setup.ps1` and `tools/antwerp/fabric/setup.ps1` merge
multiple feeders' KQL scripts into a single Eventhouse database. Today they
work only because the curated source list happens to be (almost) collision-free.
A real collision already exists: `Station` is created by both `noaa.kql` and
`waterinfo_vmm.kql` when antwerp deploys. As we add more sources to these
aggregators, every shared name (`Station`, `Vessel`, `Observation`,
`Forecast`, ãÎ) is a latent bug.

avrotize 0.x now ships `--namespace <NS>` and `--qualified-table-names` on
both `s2k` and `a2k`. With both flags, output becomes:

```
.create-merge table ['<ns>.<TableName>'] ( ... );
.create materialized-view ['<ns>.<TableName>Latest'] on table ['<ns>.<TableName>'] { ... }
.alter table ['<ns>.<TableName>'] policy update [ ... ]
ingestion json mapping "<ns>.<TableName>_json_flat"
ingestion json mapping "<ns>.<TableName>_json_ce_structured"
```

Verified locally ã bracketed-quoted form keeps KQL legal and predictable.

## Approach
Treat the namespace as a deployment-time setting controlled by
`tools/generate-kql-from-xreg.ps1`. Every source picks **exactly one** stable
namespace string derived from its repo folder (e.g. `pegelonline`,
`waterinfo_vmm`, `aisstream`). That gives clean, short, dependable table
prefixes regardless of whatever the schema author wrote into `$id` or Avro
`namespace`.

### Namespace convention
- Default: source folder name with hyphens ã underscores
  (`waterinfo-vmm` ã `waterinfo_vmm`).
- Sources may override by setting `Namespace = "..."` at the top of their
  `kql/create-kql-script.ps1` (or via env var `KQL_NAMESPACE`).
- Stored as a single token, no dots ã keeps qualified names short
  (`waterinfo_vmm.Station`, not `be.vlaanderen.waterinfo.vmm.Station`).

## Decisions (resolved with user)

1. **Namespace style: full dotted FQN matching CloudEvents type.**
   Per-source `Namespace` defaults to the schema's existing namespace
   (Avro: `namespace` field; JsonStructure: derive from `$id` path or
   declared FQN). Resulting tables look like
   `['de.wsv.pegelonline.Station']`, `['be.vlaanderen.waterinfo.vmm.Station']`,
   `['io.aisstream.PositionReport']`, etc.
2. **Roll-out: all sources in one sweep**, but each source PR must verify
   that its own shipped consumer assets (maps, dashboards, notebooks, post-
   deploy wirers) are updated to use the new names. See the consumer-touch
   list below.
3. **Aggregator hand-authored tables** (`tools/pugetsound/fabric/pugetsound.kql`,
   `tools/eurowater/fabric/kql_database.kql`) get rewritten by hand to use
   FQN-prefixed names consistent with what their source feeders produce.
4. **Migration**: redeploy creates new qualified tables side-by-side. Old
   tables stop receiving (their update policy is replaced by the new flow).
   Document `.rename table Old to ['ns.Old']` for operators who want to keep
   historical data. No auto-rename in deploy script.
