## Overview

The **Roads and Transport Authority (RTA)** is Dubai's integrated transport authority, responsible for:
- **Roads**: Sheikh Zayed Road, E11 (Abu Dhabi–Dubai highway), Dubai ring roads, 50+ major corridors
- **Metro**: Dubai Metro Red Line (52 km, 29 stations) + Green Line (23 km, 20 stations)
- **Tram**: Dubai Tram (10.6 km, 11 stops)
- **Bus**: 1,500+ buses, 119 routes
- **Taxi**: 10,000+ taxis (Hala Taxi, Dubai Taxi Corporation)
- **Marine**: Dubai Ferry, water taxis (Dubai Creek, Dubai Marina)
- **Salik**: Electronic toll collection (8 toll gates across Dubai)
- **Parking**: Smart parking systems (RTA meters, Mawaqif)

RTA has publicly stated it operates a **Smart Traffic Management System** with:
- 1,000+ traffic cameras
- 500+ speed radars
- Traffic sensors on all major roads
- Real-time incident detection
- Variable message signs (VMS)

**RTA announced in 2019** that it would publish open data through Dubai Pulse / Dubai Data. Historical documentation suggests XML feeds for:
- Traffic situations (incidents, roadworks, closures)
- Traffic speed (segment-level travel speeds)
- Parking availability (garage occupancy)
- Public transport (possibly GTFS-RT for Metro/Tram/Bus)

## Known Challenge: Web Application Firewall

**All RTA XML endpoints are protected by an aggressive WAF** (Web Application Firewall). Automated requests (curl, wget, Python requests, browser developer tools) return:

```html
<html><head><title>Request Rejected</title></head>
<body>The requested URL was rejected. Please contact RTA call center 
with below support ID (ask@rta.ae Ph:8009090).<br><br>Your support 
ID is: [random_number]<br><br><a href='javascript:history.back();'>
[Go Back]</a></body></html>
```

**This indicates**:
- RTA uses bot detection (likely Cloudflare, Akamai, or Radware)
- The endpoints exist and are the correct URLs (otherwise we'd get HTTP 404 instead of WAF rejection)
- The WAF may allow browsers but blocks HTTP clients without JavaScript / cookies / TLS fingerprints
- The data may be intended for internal systems or partner applications, not public API access

**Why this is frustrating**:
- The URLs are well-known and documented in older RTA press releases and third-party apps
- The data is likely generated by RTA's traffic management system (it exists)
- The WAF is blocking legitimate API access that RTA itself advertised as "open data"
