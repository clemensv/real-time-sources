# Malaysia APIMS/MyEQMS — Air Pollution Index System

**Country/Region**: Malaysia
**Publisher**: Department of Environment (DOE), Ministry of Natural Resources and Environmental Sustainability
**API Endpoint**: `https://apims.doe.gov.my/` (MyEQMS portal — Quasar SPA)
**Documentation**: https://apims.doe.gov.my/
**Protocol**: Web SPA (Quasar/Vue.js framework); backend API undocumented
**Auth**: N/A (SPA publicly accessible; API endpoints undocumented)
**Data Format**: SPA renders JSON from internal API
**Update Frequency**: Hourly API (Air Pollutant Index updated hourly)
**License**: Malaysian government data

## What It Provides

Malaysia's Air Pollutant Index Monitoring System (APIMS), now rebranded as MyEQMS (My Environmental Quality Monitoring System), monitors air quality across Malaysia's 13 states and 3 federal territories.

Key data:
- **Air Pollutant Index (API)**: Malaysia's national air quality index (0-500+ scale)
- **Pollutants**: PM2.5, PM10, SO₂, NO₂, CO, O₃
- **Station network**: ~65 continuous air quality monitoring stations
- **Water quality**: MyEQMS also covers river water quality monitoring
- **Haze monitoring**: Critical during transboundary haze events (Indonesian fire smoke)

### Regional Significance

Malaysia's air quality monitoring is particularly important for:
- **Transboundary haze**: Indonesian peat/forest fire smoke regularly blankets Malaysia (2015, 2019 crises)
- **Southeast Asian air quality corridor**: Singapore, Malaysia, and Indonesia share an airshed
- **Industrial monitoring**: Petrochemical zones in Johor, Terengganu

### Probe Results

The MyEQMS portal at `apims.doe.gov.my` loads as a Quasar (Vue.js) SPA. The application makes XHR requests to internal API endpoints, but these are not publicly documented.

Attempts to access backend API endpoints:
- `/api/general/station?type=air` → Returns SPA HTML (not API response)
- `/api/general/stations` → Returns SPA HTML
- `api.myeqms.doe.gov.my` → Connection failed

The SPA catches all routes and returns the application shell. The actual API is either on a different subdomain or requires authentication tokens generated by the SPA.

### Malaysia Open Data Portal (data.gov.my)

Malaysia's open data portal at `data.gov.my` is a modern Next.js application (open source on GitHub!). However, searching for "air quality" returned no entries during testing. The portal focuses on statistical/census data rather than real-time environmental monitoring.

## Entity Model

- **Station**: Named monitoring stations (e.g., "Shah Alam", "Johor Bahru") with coordinates
- **API Index**: Hourly composite index (0-500+) with category (Good, Moderate, Unhealthy, etc.)
- **Pollutant**: Individual pollutant concentrations
- **State**: Malaysian state/territory

## Integration Notes

- The SPA's backend API could potentially be reverse-engineered from browser network requests
- Malaysia's open data portal (data.gov.my) source code is on GitHub — could reveal API patterns
- OpenAQ may aggregate some Malaysian station data
- Transboundary haze events make this data regionally critical
- Consider combining with Singapore NEA data (already in repo) for SE Asian air quality corridor
- The ASEAN Agreement on Transboundary Haze Pollution may have data sharing implications

## Verdict

Important regional source hidden behind an SPA. Malaysia's air quality data is critical for Southeast Asian transboundary haze monitoring but the MyEQMS portal doesn't expose its API. The modernized Quasar SPA indicates active development — a public API may be forthcoming. The open-source data.gov.my portal suggests Malaysia is moving toward open data. Worth revisiting.
