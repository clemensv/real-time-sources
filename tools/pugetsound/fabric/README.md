# Fabric Event Stream + KQL Database Setup

Sets up a Microsoft Fabric Event Stream and KQL database for raw Puget Sound
CloudEvents landing.

## Architecture

```
┌──────────────────────────────────────────────────────────────────────┐
│ ACI Container Group / Docker Compose                                │
│  seattle-911                                                        │
│  seattle-street-closures                                            │
│  king-county-marine                                                 │
│  epa-uv                                                             │
│  nws-forecasts                                                      │
│  wsdot                                                              │
│  noaa --station 9444090,9444900,9445958,9446484,                    │
│       9447130,9449424,9449880                                       │
└────────────────────────────────────┬─────────────────────────────────┘
                                     │ CONNECTION_STRING
                                     ▼
┌──────────────────────────────────────────────────────────────────────┐
│ Fabric Event Stream: pugetsound-live-ingest                         │
│  Source: pugetsound-live-input (Custom Endpoint)                    │
│  Stream: pugetsound-live-ingest-stream                              │
│  Destination: _cloudevents_dispatch                                 │
└────────────────────────────────────┬─────────────────────────────────┘
                                     ▼
┌──────────────────────────────────────────────────────────────────────┐
│ Eventhouse / KQL database: pugetsound-live                          │
│  _cloudevents_dispatch                                               │
│  All typed tables + latest views, fully namespace-qualified         │
│  (Seattle, King County, EPA UV, NWS, WSDOT, NOAA)                   │
└──────────────────────────────────────────────────────────────────────┘
```

All typed tables use fully namespace-qualified names that match each
feeder's own KQL script (for example `['US.WA.Seattle.Fire911.Incident']`
and `['Microsoft.OpenData.US.NOAA.WaterLevel']`), so the shared Eventhouse
has no table-name collisions between sources.

There is no SQL normalization layer here. The Event Stream lands structured
CloudEvents as-is into `_cloudevents_dispatch`, and KQL update policies fan
them back out into typed tables.

## Files

| File | Description |
|---|---|
| `setup.ps1` | Creates or updates the KQL database and Event Stream |
| `pugetsound.kql` | Combined KQL for all sources (Seattle 911, Seattle street closures, King County marine, EPA UV, NWS forecasts, WSDOT, NOAA) with fully qualified table names, rebuilt from each feeder's `kql/*.kql` script |

`pugetsound.kql` is regenerated by concatenating the per-feeder KQL scripts
under `../../../feeders/<source>/kql/` and de-duplicating the shared
`_cloudevents_dispatch` landing table. Each feeder's script is itself
generated from its xRegistry contract via `tools/generate-kql-from-xreg.ps1`
with qualified table names.

## Usage

```powershell
./setup.ps1 `
  -Workspace "c98acd97-4363-4296-8323-b6ab21e53903" `
  -Eventhouse "dbfd2819-2879-4ae7-bff2-95619ad7b8e7"
```

`-Workspace` and `-Eventhouse` each accept either a Fabric display name or a
GUID. For example, you can pass friendly names instead:

```powershell
./setup.ps1 -Workspace "Puget Sound" -Eventhouse "pugetsound-live"
```

This creates or updates:

- KQL database `pugetsound-live`
- Event Stream `pugetsound-live-ingest`
- Custom endpoint source `pugetsound-live-input`
- Eventhouse destination landing into `_cloudevents_dispatch`

After setup, open the Event Stream in the Fabric portal and retrieve the custom
endpoint connection string for the container group or Docker Compose bundle.

## Example queries

```kusto
_cloudevents_dispatch
| summarize count() by type
| order by count_ desc

['US.WA.Seattle.Fire911.IncidentLatest']
| top 20 by ___time desc
| project incident_number, incident_type, incident_datetime, address

['us.wa.seattle.streetclosures.StreetClosureLatest']
| where end_date >= format_datetime(now(), "yyyy-MM-dd")
| project closure_id, permit_type, project_name, street_on, street_from, street_to

['us.wa.kingcounty.marine.WaterQualityReadingLatest']
| project station_name, observation_time, water_temperature_c, dissolved_oxygen_mg_l, ph

['us.epa.uvindex.DailyForecastLatest']
| project location_id, forecast_date, uv_index, uv_alert

['Microsoft.OpenData.US.NOAA.NWS.Forecasts.LandZoneForecastLatest']
| mv-expand periods
| project zone_id, updated, periods

['Microsoft.OpenData.US.NOAA.NWS.Forecasts.MarineZoneForecastLatest']
| project zone_id, zone_name, issued_at_text, synopsis

['us.wa.wsdot.ferries.VesselLocationLatest']
| top 20 by ___time desc

['Microsoft.OpenData.US.NOAA.WaterLevelLatest']
| top 20 by ___time desc
| project station_id, timestamp, value
```
